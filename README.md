# Institute of Digital Risk (IDR) — Assignment Portfolio

A collection of design and development assignments produced for the **Institute of Digital Risk**, a tech-education brand focused on training, deployment, and innovation in digital risk management.

---

## Project Structure

```
assignment-1/
└── IDR_Logo_Design_Note.txt   — Design rationale for the IDR brand logo

assignment-2/
└── IDR_Homepage.html          — Fully coded homepage for the IDR website
```

---

## Assignment 1 — Logo Design

**File:** `assignment-1/IDR_Logo_Design_Note.txt`

Documents the design decisions behind the IDR brand identity:

- **Icon:** A cube-inspired geometric mark representing structure, layered risk, and resilience, featuring balanced faces and a reinforced outer frame.
- **Colour palette:** Orange (`#FF6B1A`) as the primary accent to convey energy, alertness, and proactive risk awareness; black and white for a professional, high-contrast identity.
- **Typography:** Clean and modern typeface with a clear hierarchy between the institution label and the core name "Digital Risk."
- **Scalability:** The icon is intentionally simple and bold to remain legible at favicon and mobile-header sizes.

## Assignment 2 — Homepage

**File:** `assignment-2/IDR_Homepage.html`

A single-file, fully self-contained homepage built with semantic HTML5 and vanilla CSS.

### Key Features

| Feature | Details |
|---|---|
| **Fonts** | Barlow Condensed (display), Barlow (body), DM Mono (code/mono) via Google Fonts |
| **Colour system** | CSS custom properties (`--orange`, `--black`, `--surface`, etc.) for consistent theming |
| **Navigation** | Fixed sticky nav with blur backdrop, scroll-aware border, animated underline links, and a hamburger menu for mobile |
| **Custom cursor** | Orange dot + ring cursor that reacts to hover states |
| **Accessibility** | Skip-to-content link, `focus-visible` outlines, semantic landmark elements |
| **Responsive** | Mobile-first layout using CSS media queries |

### How to View

Open `assignment-2/IDR_Homepage.html` directly in any modern web browser — no build step or server required.
